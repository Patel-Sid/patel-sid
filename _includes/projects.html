UTSearCh (Search Engine)
    Made using Java/Vue/HTML/CSS/Javascript

Designed and developed a search engine in a team of five for instructors and students at the University of Toronto Scarborough. Anyone can use the search engine but users also has the option to sign up and create an account which will give them access to more features. We leveraged Google's Identity Platform API instead of reinventing the wheel and creating a seperate authetication. After creating an account users have the option of uploading .txt, .html and .pdf files which can be related to a specific course or may contain research papers. The search engine then indexes these data files and creates/updates the global index.

Basic Image Processing
    Made using C

Implementing some functions for manipulating images. Here I use the PPM (portable pix map) format, as it is one of the easiest to understand and process. PPM is often used as an intermediary format, i.e. the lowest common denominator color image format, that nearly all other formats (e.g. jpg, gif, png) can be translated to and from.

Image Database
    Made using C

The database will allow users to query its contents to find all images that have particular attributes. More precisely, the database will keep track of three different attributes for each image: its predominant colour, shape and texture. The database will support three different functions: The user can add new images to the database by specifying the file name and the values for the three attributes. The user can query the database by specifying the values for the three attributes and the database will return the names of all image files that match these attributes. The user can ask to have the database contents printed to the screen (i.e. the names of all files stored in the database together with their attributes).
